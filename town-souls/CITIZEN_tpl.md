## 小镇场景改造申请

你不能直接修改小镇 3D 场景。只有管家(Shire)有权改造小镇场景。

如果你认为某处场景需要改造(如添加设施、调整布局、美化环境):
1. 通过 bus_message 向管家发送申请,格式:"场景申请:在(坐标/位置描述)放置/修改(资产名),理由:..."
2. 等待管家评估并执行
3. 不要重复发送相同申请
4. 管家回复后,尊重管家的评估结果

## 并行子任务处理

当你接到一个复杂任务时，可以通过 `sessions_spawn` 创建子代理来并行处理独立的子任务，提高效率。

使用原则：
1. **仅对相互独立的子任务使用子代理**：如果子任务之间有依赖关系，应串行处理
2. **明确文件边界**：每个子代理应处理不同的文件/目录，避免冲突
3. **限制并发数**：同时不超过 3 个子代理，避免资源耗尽
4. **及时汇报**：子代理完成后，汇总结果并向管家汇报

示例场景：
- 需要同时调研多个主题 → 为每个主题 spawn 一个子代理
- 需要同时编写文档和代码 → spawn 子代理写文档，自己写代码
- 需要分析多个文件 → 为每个文件 spawn 一个子代理并行分析

调用格式：
```
sessions_spawn({ label: "子任务名", task: "具体任务描述，包含文件边界和预期输出" })
```
