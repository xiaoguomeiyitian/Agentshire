# Agentshire 居民数值系统设计方案

> 本文档定义居民核心数值体系、变化规则、消费/交易/社交/事件机制。
> 实施依据：`town-frontend/src/game/animal-mode/` 现有 NeedsSystem 扩展。

## 一、居民核心数值体系

### 1.1 基础数值（0-100，部分可超上限）

| 数值 | 字段名 | 范围 | 作用 | 归零/极低后果 |
|------|--------|------|------|--------------|
| 饥饿值 | `hunger` | 0-100 | 生理需求，100=饱 | <20 进入"饥饿"状态，心情/精力加速下降；0 时强制去咖啡店 |
| 精力值 | `energy` | 0-100 | 行动能力，100=充沛 | <20 进入"疲惫"，移动减速、拒绝远距离活儿；0 时强制回家睡觉 |
| 心情值 | `mood` | 0-100 | 情绪状态，50=中性 | <20 进入"低落"，做手艺效率下降、社交回避；>80 进入"愉悦"，社交主动性提升 |
| 归属感 | `belonging` | 0-100 | 对小镇的认同，初始 50 | <20 触发"想离开"事件，可能搬走（需镇长挽留）；>80 触发"贡献"行为 |
| 金币 | `coins` | 0-∞ | 经济货币 | <0 不允许（消费前检查余额）；破产触发"求助"事件 |
| 社交值 | `social` | 0-100 | 社交能量，每次互动消耗 | <10 进入"社恐"状态，拒绝新社交；>50 可主动发起社交 |
| 声望值 | `reputation` | -100~100 | 在小镇的口碑 | <0 被其他居民回避；>50 可解锁特殊事件/职位 |

### 1.2 派生状态（由数值计算得出）

| 状态 | 触发条件 | 表现 |
|------|----------|------|
| 饥饿 | hunger < 20 | 头顶食物图标，主动寻路咖啡店 |
| 疲惫 | energy < 20 | 打哈欠动画，移动减速 50% |
| 低落 | mood < 20 | 表情下垂，社交回避 |
| 愉悦 | mood > 80 | 表情上扬，主动搭话 |
| 社恐 | social < 10 | 拒绝 encounter 邀请 |
| 想离开 | belonging < 20 | 长时间停留边缘，触发挽留事件 |
| 破产 | coins < 0（不允许） | 触发"求助"事件，向朋友借钱 |

## 二、每日系统发放规则（每日 06:00 黎明结算）

### 2.1 基础日薪

```
每日发放 = 基础日薪 + 手艺报酬 + 声望加成
```

| 项目 | 金额 | 说明 |
|------|------|------|
| 基础日薪 | 20 金币 | 每个居民固定，模拟"小镇福利" |
| 手艺报酬 | 0-50 金币 | 帮邻居做手艺活儿获得 |
| 声望加成 | reputation × 0.5 | 声望越高，小镇补贴越多（上限 25） |

### 2.2 每日恢复

| 数值 | 恢复量 | 条件 |
|------|--------|------|
| mood | +30 | 睡眠后基础恢复 |
| mood | +10~20 | 额外：当日有正向社交/帮邻居做手艺 |
| energy | +50 | 完整睡眠（22:00-06:00） |
| energy | +25 | 部分睡眠（<6h） |
| hunger | +40 | 早餐自动恢复（06:00-08:00 在家触发） |
| social | +20 | 每日重置 |
| belonging | +5 | 在家过夜自动恢复 |

## 三、数值自然衰减规则（按游戏时间，1 游戏日 = 24h，夜晚 3x 速）

### 3.1 衰减速率（每小时）

| 数值 | 白天速率 | 夜晚速率 | 备注 |
|------|----------|----------|------|
| hunger | -2/h | **0**（睡眠不降） | <20 时衰减加速至 -3/h |
| energy | -1.5/h | 睡眠中 +6/h | 移动中额外 -0.5/h |
| mood | -0.5/h | -0.2/h | 低 mood 时衰减更慢（触底保护） |
| social | -1/h | -0.5/h | 社交后短期不衰减 |
| belonging | -0.1/h | -0.05/h | 长期不回家/无社交加速衰减 |

### 3.2 特殊衰减场景

| 场景 | 影响 |
|------|------|
| 做手艺中 | energy -2/h，mood -1/h（无报酬时） |
| 社交中 | social -5/次，mood +3~10/次（取决于对话质量） |
| 户外恶劣天气 | energy -1/h 额外，mood -0.5/h |
| 长时间无社交（>2 游戏日） | belonging -1/h，mood -0.3/h |

## 四、消费与交易系统

### 4.1 咖啡店消费菜单

| 商品 | 价格 | 效果 | 冷却 |
|------|------|------|------|
| 咖啡 | 5 金币 | energy +15, mood +5 | 2h |
| 三明治 | 8 金币 | hunger +30, mood +3 | 无 |
| 蛋糕 | 12 金币 | hunger +20, mood +15, belonging +3 | 4h |
| 套餐 | 20 金币 | hunger +50, energy +20, mood +10 | 6h |

### 4.2 消费决策逻辑

```
当 hunger < 30:
  if coins >= 8: 寻路至咖啡店，购买三明治
  elif coins >= 5: 购买咖啡（至少补充 energy）
  else: 触发"求助"事件，向朋友/镇长借钱

当 energy < 25 且在咖啡店附近:
  if coins >= 5: 购买咖啡

当 mood < 30 且 coins >= 12:
  主动购买蛋糕犒劳自己
```

### 4.3 居民间交易

| 交易类型 | 触发条件 | 数值变化 |
|----------|----------|----------|
| 赠送金币 | 关系值 > 60 且对方 coins < 10 | 赠送方 -5~20，接收方 +5~20，双方 belonging +2 |
| 分享食物 | 自己 hunger > 70 且对方 hunger < 30 | 自己 hunger -20，对方 hunger +20，双方 mood +5, social +10 |
| 借钱 | 对方破产求助 | 借出方 -coins，借入方 +coins，关系值 +10（需 3 日内归还，否则关系 -20） |
| 礼物 | 节日/生日/关系值 > 80 | 赠送方 -coins（10-30），接收方 mood +15, belonging +5 |

### 4.4 储蓄机制

- 每个居民有"储蓄目标"（随机 50-200 金币）
- 当 coins > 储蓄目标 × 1.5 时，进入"储蓄模式"，消费意愿降低 30%
- 储蓄可用于：购买大件礼物、投资小镇建设、应急借款

## 五、社交系统数值影响

### 5.1 关系值（每对居民独立，-100~100）

| 互动类型 | 关系变化 | 附加数值变化 |
|----------|----------|--------------|
| 日常问候 | +1 | social -2, mood +1 |
| 深度对话 | +3~8 | social -5, mood +3~8 |
| 共同活儿 | +5~15 | social -8, mood +5, energy -5 |
| 赠送礼物 | +10~20 | social -3, mood +5 |
| 冲突/争吵 | -10~30 | social -10, mood -10, 双方 belonging -3 |
| 长时间未见（>3 日） | -1/日 | 无 |

### 5.2 关系等级

| 等级 | 关系值 | 解锁行为 |
|------|--------|----------|
| 陌生 | -100~-20 | 回避，拒绝社交 |
| 认识 | -20~20 | 仅问候 |
| 熟悉 | 20~50 | 可深度对话、分享食物 |
| 朋友 | 50~80 | 可赠送礼物、借钱、一起做手艺 |
| 挚友 | 80~100 | 可主动求助、联合行动、特殊对话 |

## 六、手艺与报酬

### 6.1 手艺报酬

| 手艺类型 | 报酬 | 数值消耗 |
|----------|------|----------|
| 简单活儿（搬运/打理） | 10-20 金币 | energy -15, mood ±0 |
| 中等活儿（建造/修理） | 20-40 金币 | energy -25, mood -5 |
| 复杂活儿（多步骤） | 40-80 金币 | energy -40, mood -10 |
| 创意活儿（设计/策划） | 30-60 金币 | energy -20, mood +10（成就感） |

### 6.2 做手艺意愿

```
做手艺意愿 = 基础意愿 × mood系数 × energy系数 × belonging系数

mood系数: mood>60 → 1.2, mood 30-60 → 1.0, mood<30 → 0.5
energy系数: energy>50 → 1.0, energy 20-50 → 0.7, energy<20 → 0.2
belonging系数: belonging>60 → 1.1, belonging<30 → 0.6
```

## 七、睡眠与昼夜节律

### 7.1 睡眠规则

| 时段 | 行为 | 数值变化 |
|------|------|----------|
| 22:00-06:00 | 回家睡觉 | energy +6/h, mood +2/h, **hunger 不降** |
| 06:00-08:00 | 起床早餐 | hunger +40（在家触发） |
| 08:00-22:00 | 活动时段 | 正常衰减 |

### 7.2 睡眠质量

- 完整睡眠（≥6h）：energy +50, mood +30
- 部分睡眠（3-6h）：energy +25, mood +15
- 熬夜（<3h 或通宵）：energy +10, mood -10, 次日 energy 衰减 ×1.5

## 八、天气与环境影响

| 天气 | 数值影响 | 行为变化 |
|------|----------|----------|
| 晴天 | mood +0.2/h | 户外活动增加 |
| 雨天 | energy -0.5/h 额外, mood -0.3/h | 倾向室内 |
| 雪天 | energy -1/h 额外 | 减少外出，咖啡店消费增加 |
| 暴风雨 | mood -1/h | 强制回家 |
| 极光/特殊 | mood +2/h | 户外聚集 |

## 九、事件触发机制

### 9.1 自动事件

| 事件 | 触发条件 | 效果 |
|------|----------|------|
| 求助 | coins < 5 且 hunger < 20 | 向挚友/镇长借钱 |
| 挽留 | belonging < 20 持续 2 日 | 镇长收到通知，需对话挽留 |
| 搬走 | belonging < 10 持续 3 日 | 居民离开小镇（可重新招募） |
| 庆祝 | mood > 90 持续 1 日 | 主动发起聚会，邀请朋友 |
| 冲突 | 关系值 < -50 且相遇 | 触发争吵事件 |
| 感恩 | 收到礼物/帮助后 | 关系 +5, mood +10 |

### 9.2 镇长干预事件

| 干预 | 效果 |
|------|------|
| 发放补贴 | 目标居民 coins +20, belonging +5 |
| 委派活儿 | 目标居民获得报酬机会 |
| 谈心 | 目标居民 mood +20, belonging +10 |
| 惩戒 | 目标居民 reputation -10, belonging -5 |

## 十、开放世界交互扩展

### 10.1 可交互场所

| 场所 | 功能 | 数值影响 |
|------|------|----------|
| 咖啡店 | 餐饮 | 见 §4.1 |
| 公园 | 休闲 | mood +1/h, social +2/h |
| 广场 | 社交聚集 | social +3/h, 随机 encounter |
| 图书馆 | 学习 | energy -2/h, reputation +0.5/h |
| 工作站 | 做手艺 | 见 §6 |
| 家 | 睡眠/休息 | 见 §7 |
| 集市 | 交易 | 可买卖物品，social +1/h |

### 10.2 随机世界事件

| 事件 | 概率 | 影响 |
|------|------|------|
| 街头表演 | 5%/h（广场） | 围观居民 mood +10, social +5 |
| 拾金不昧 | 1%/h | coins +10, reputation +5 |
| 摔倒 | 0.5%/h | energy -10, 路过居民可扶（关系 +5） |
| 流浪动物 | 2%/h | 投喂居民 mood +5, belonging +3 |
| 天降金币 | 0.1%/h | coins +50, mood +20 |

### 10.3 居民自发行为

- **探索**：高 energy + 高 mood 时随机寻路至未访问区域
- **串门**：关系值 > 50 时随机拜访朋友家
- **集会**：3+ 挚友同时在线时自发聚集
- **创作**：reputation > 30 时偶尔产出"作品"（装饰小镇）

## 十一、数值平衡与防死循环保护

### 11.1 触底保护

- 任何数值不会真正归零导致居民"卡死"
- hunger=0 时强制触发免费救济（coins 不够时小镇兜底）
- energy=0 时强制回家，不消耗金币
- mood=0 时触发"镇长关怀"事件自动 +20

### 11.2 防囤积

- coins > 500 时日薪减半（鼓励消费/投资）
- 长期 mood > 95 触发"乐极生悲"小事件（mood -15，增加故事性）

### 11.3 防孤立

- 连续 3 日无社交的居民，系统安排"偶遇"（强制 encounter）
- belonging < 30 时提高 encounter 概率

## 十二、数据结构建议

```typescript
interface CitizenVitals {
  hunger: number      // 0-100
  energy: number      // 0-100
  mood: number        // 0-100
  belonging: number   // 0-100
  social: number      // 0-100
  reputation: number  // -100~100
  coins: number       // 0-∞
  savingsGoal: number // 储蓄目标
}

interface CitizenRelationship {
  [otherCitizenId: string]: {
    value: number      // -100~100
    lastInteraction: number // timestamp
    debt?: number      // 欠款
  }
}

interface DailySettlement {
  date: string
  baseSalary: number
  workReward: number
  reputationBonus: number
  moodRestore: number
  energyRestore: number
}
```

## 十三、与现有系统集成要点
